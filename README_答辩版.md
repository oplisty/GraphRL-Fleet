# Data-Structure-HW（答辩版说明）

> 课程主题：新能源物流车队协同调度  
> 目标：围绕课程大作业要求，完成一个具备**图结构建模、动态任务仿真、路径规划、调度策略、充电约束、实验输出与可视化展示**的完整系统。

---

## 1. 项目背景与问题定义

本项目针对如下场景：

- 中央仓库管理一支数量有限的新能源配送车队
- 城市中不断动态出现新的运输任务
- 每辆车具有电量上限、载重上限、速度、耗电参数等约束
- 车辆执行任务时需要规划路径，并在必要时前往充电站补能
- 充电站还存在队列和负荷压力，不是简单“到站即充”
- 希望通过不同调度策略对比，分析哪种方法在该问题上更有效

本项目本质上是一个**带资源约束的动态图调度仿真问题**。

## 2. 对课程要求的完成情况

根据 `2026-大作业要求.txt`，本仓库当前完成情况如下。

### 需求 1：有限车辆 + 电量上限 + 载重上限 + 动态任务
**完成情况：已完成**

- 车队数量有限，可配置
- 车辆具备：
  - 电量上限
  - 当前电量
  - 载重上限
  - 当前所在节点

**完成情况：已完成**

任务包含：
- 产生时间 `release_time`
- 目标位置（通过图节点表示）
- 货物重量 `weight`
- 截止时间 `deadline`

这些任务由生成器随机生成，并在仿真主循环中逐步进入可调度状态。

### 需求 3：越早完成且路径越短收益越高，超时扣分
**完成情况：已完成**

系统在仿真过程中对任务进行评分，综合考虑：
- 实际执行距离
- 等待时间
- 是否超时

超时任务会转为过期状态，并产生惩罚。

### 需求 4：电量不足时寻找最近或最优充电站
**完成情况：已完成**

在派单与执行阶段，系统都会判断车辆是否：
- 能到达任务点
- 能完成任务后回仓
- 若不能，则转向充电站补能

选站时不仅看距离，也综合考虑站点负载情况。

### 需求 5：考虑充电站排队与负荷压力
**完成情况：已完成**

充电站包含：
- 充电桩数量
- 已占用桩位
- 排队队列
- 等待车辆

仿真会更新：
- 到站排队
- 开始充电
- 结束充电
- 出站继续运行

### 需求 6：多车协同完成同一任务
**完成情况：已完成（支持扩展）**

系统支持协同任务建模：
- 一个任务可由多辆车分担运输重量

- 默认基线实验通常仍采用单车单任务，以便做清晰对比

## 3. 使用了哪些核心数据结构与算法

## 3.1 图结构建模

道路网络使用图表示：
- **节点**：道路交汇点、仓库、任务位置、充电站位置等
- **边**：道路连接关系，边上带有距离等权重信息

优势：
- 能自然表达城市道路网络
- 易于接入最短路与启发式搜索算法
- 方便在仿真过程中做可达性判断

---

## 3.2 路径规划算法

当前已支持三种路径规划方式：

### 1) Dijkstra
- 用于带权图最短路径
- 适合作为稳定、标准、可解释的基线算法
- 在本项目中用于最短距离与最短路径查询

### 2) A*
- 在 Dijkstra 基础上加入启发函数
- 能减少搜索范围，提高面向目标点查询时的效率
- 适合作为更“有目标导向”的路径搜索方法

### 3) RRT
- 属于采样式路径搜索思路
- 更偏实验性和扩展性展示
- 可用于说明项目在算法上具备进一步研究与拓展空间

---

## 3.3 调度策略

### 最近任务优先
思路：
- 空闲车辆优先选择距离当前最近的任务

特点：
- 实现简单
- 路径代价较低
- 适合作为基础策略

### 最大任务优先
思路：
- 优先调度重量更大的任务

特点：
- 更关注高负载任务处理
- 可与最近优先形成明确对比

---

## 3.4 状态管理与仿真机制

系统在时间步推进下运行，核心机制包括：
- 任务按 `release_time` 动态释放
- 调度器在每个时间步做派单决策
- 车辆沿图路径移动
- 电量随里程减少
- 必要时进入充电站排队与充电
- 到达任务点后完成服务并累计收益
- 超时任务转为过期

这相当于一个**离散时间仿真系统**。

## 4. 项目创新点与特色

相对于只完成“基础最短路 + 派单”的作业，本项目有以下亮点：

### 1) 不是单点算法，而是完整仿真闭环
项目同时覆盖：
- 任务生成
- 图建模
- 路径规划
- 调度
- 充电约束
- 日志输出
- 可视化展示

形成了较完整的系统工程闭环。

### 2) 引入新能源特有约束
与普通物流调度不同，本项目加入了：
- 电量限制
- 回仓可达性检查
- 最近/最优充电站选择
- 充电站排队与负荷

更贴近新能源物流场景。

### 3) 支持动态任务而非静态任务集
系统不是一次性给出所有任务，而是根据时间逐步出现任务，模拟真实运行环境中的不确定性。

### 4) 可扩展性较强
仓库结构已经为后续扩展预留空间：
- 可更换调度器
- 可替换路径规划算法
- 可接入 RL 或离线优化策略
- 可接入前端进行实时展示

### 5) 提供前后端联动展示
系统不仅有算法和引擎，还能通过 Web UI 进行直观展示，有助于答辩演示和结果说明。

- 地图和节点展示
- 车辆、任务、充电站状态展示
- 统计面板和事件日志展示
